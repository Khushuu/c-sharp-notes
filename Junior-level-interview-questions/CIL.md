## Common Intermediate Language (CIL)

### What is the Common Intermediate Language?

- **CIL (Common Intermediate Language)** is the intermediate programming language used by the .NET platform.
- .NET-compatible languages such as **C#**, **Visual Basic**, and **F#** are compiled into CIL rather than directly into machine code. (binary code)
- The CIL code is stored in assemblies like **.dll** or **.exe** files.

### How CIL fits into the .NET compilation and execution pipeline

1. Write source code in a .NET language (for example, C#, Visual Basic, or F#).
2. The language compiler compiles the source code into **CIL**.
3. When the application runs, the **Common Language Runtime (CLR)** uses the **Just-In-Time (JIT) compiler** to translate CIL into **binary machine code**.
4. The machine code is executed by the processor.

### Why CIL is important

- CIL enables **language interoperability** across the .NET ecosystem.
- Because all .NET languages compile to the same intermediate representation, a class written in one language can inherit from or use a class written in another language.
- This shared intermediate language is a key reason why .NET supports mixed-language solutions.

### Example workflow to view CIL

- Build your project in Visual Studio.
- Locate the generated assembly, usually in a folder such as `bin/Debug/net5.0`.
- Use **ILDASM** (Intermediate Language Disassembler) to inspect the CIL code.
  - ILDASM is installed with .NET tools.
  - Example path: `C:\Program Files (x86)\Microsoft SDKs\Windows\v10.0A\bin\NETFX 4.8 Tools\ildasm.exe`.
- Drag the compiled `.dll` into ILDASM and inspect the methods, such as `Main`.

### Key definitions

- **CIL**: Common Intermediate Language, the intermediate code form generated by .NET language compilers.
- **CLR**: Common Language Runtime, the .NET runtime environment that executes managed code.
- **JIT compiler**: Just-In-Time compiler that converts CIL into native machine code at runtime.
- **Assembly**: A compiled .NET output file (`.dll` or `.exe`) that contains CIL and metadata.

### Important interview questions

- **How is it possible that a C# class can derive from an F# class?**
  - Because both C# and F# compile to the same intermediate language, CIL, enabling interoperability.

- **Does the C# compiler compile source code directly to binary code?**
  - No. The C# compiler compiles the code to CIL first. The CLR then uses the JIT compiler to turn CIL into binary machine code at runtime.

- **How can you view the CIL code a project was compiled to?**
  - Use tools such as **ILDASM** to open the assembly and inspect its CIL instructions.

- **What is the Just-In-Time compiler?**
  - The JIT compiler is a CLR feature that converts CIL code into native binary code while the program is executing.
