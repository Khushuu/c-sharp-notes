# Choosing ASP.NET Core

## Should you use it?
- You now have a general grasp of what ASP.NET Core is and the types of applications you can build.
- The key question is: **should you use it?**
- **Microsoft recommends** that all new .NET web development use ASP.NET Core.
- Switching to or learning a new web stack is a big ask for any developer or company.

## Why choose ASP.NET Core?
- If you’re new to .NET development and considering ASP.NET Core:
  - Microsoft positions it as an attractive option for beginners
  - .NET cross-platform support means it competes with many other frameworks on their own turf
- ASP.NET Core selling points:
  - **modern, high-performance, open-source web framework**
  - uses **familiar design patterns and paradigms**
  - supports **C#** (also **VB.NET** or **F#** if preferred)
  - can build and run on **any platform**

## Platform support
- ASP.NET Core is a reimagining of the ASP.NET framework.
- Built with modern software design principles on top of the new .NET platform.
- .NET (previously called .NET Core) has had widespread production use since 2016.
- ASP.NET Core draws from the mature and reliable **.NET Framework**.
- Choosing ASP.NET Core and **.NET 7** gives:
  - a **dependable platform**
  - a **full-featured web framework**

## Cross-platform benefits
- Major selling point: **develop and run on any platform**.
- ASP.NET Core apps run on **Mac, Windows, and Linux**.
- Supported Linux distributions include:
  - **RHEL**
  - **Ubuntu**
  - **Debian**
  - **CentOS**
  - **Fedora**
  - **openSUSE**
- ASP.NET Core also runs on **Alpine** for compact container deployments.
- This means more operating system options are viable.

## Existing .NET developers
- The decision to invest in ASP.NET Core was largely a question of timing.
- Early versions of .NET Core lacked some adoption features.
- Latest versions of .NET no longer have that problem.
- Microsoft now explicitly advises that all new .NET applications should use **.NET 7 or newer**.

## Legacy framework support
- Microsoft continues bug and security fixes for the older ASP.NET framework.
- No more feature updates will be provided for the older framework.
- **.NET Framework is not being removed.**
- Old applications will continue to work.
- New development should use **ASP.NET Core / .NET 7** instead of .NET Framework.

## Main benefits vs previous ASP.NET
- Cross-platform development and deployment
- Performance as a core feature
- Simplified hosting model
- Regular releases with a shorter cycle
- Open-source model
- Modular features
- More application paradigm options
- Ability to package .NET with an app for standalone deployments

## New possibilities
- Cross-platform deployment opens new application possibilities:
  - cheaper Linux VM hosting in the cloud
  - Docker containers for repeatable CI
  - writing .NET code on Mac without a Windows VM
- ASP.NET Core with .NET 7 makes these possibilities viable.

## Transition notes
- ASP.NET Core retains many concepts from the previous ASP.NET framework.
- You can still run ASP.NET Core applications in **IIS**.
- Moving to ASP.NET Core does not mean starting from scratch.
