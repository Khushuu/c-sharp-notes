# What You’ll Learn in This Book

## Book focus
- In-depth tour of the **ASP.NET Core framework**.
- Practical guide to real-world ASP.NET Core development.

## Recommended background
- Familiarity with **C#** or a similar object-oriented language.
- Basic familiarity with **HTML** and **JavaScript**.

## Learning goals
- Learn how to build **HTTP API applications** using **minimal APIs**.
- Learn how to create **page-based applications** with **Razor Pages**.
- Understand key ASP.NET Core concepts such as **model-binding**, **validation**, and **routing**.
- Learn to generate HTML using **Razor syntax** and **Tag Helpers**.
- Learn to use **dependency injection**, **configuration**, and **logging** as applications grow.
- Learn to protect applications using **security best practices**.

## Example approach
- Examples are generally **small and self-contained**.
- Designed to focus on a **single feature at a time**.
- Helps you learn and explore each concept clearly.

## Development environment
- Most examples use **Visual Studio**.
- You can follow along using any editor or IDE.
- Appendix A includes details for:
  - setting up your editor or IDE
  - installing the **.NET 7 SDK**
- Examples shown with Windows tools also apply to **Linux** and **macOS**.

## Tip
- Install **.NET 7** from https://dotnet.microsoft.com/download.
- Appendix A includes more details on configuring your environment for **ASP.NET Core** and **.NET 7**.

## Summary
- **ASP.NET Core** is a **cross-platform, open-source, high-performance web framework**.
- ASP.NET Core runs on **.NET**, previously called **.NET Core**.
- Use **Razor Pages** or **MVC controllers** for server-rendered, page-based applications.
- Use **minimal APIs** or **Web APIs** for RESTful or HTTP APIs.
- Use **gRPC** for highly efficient server-to-server RPC applications.
- Use **Blazor WebAssembly** for browser-based client applications.
- Use **Blazor Server** for stateful server-rendered applications with WebSocket updates.
- Microsoft recommends **ASP.NET Core** and **.NET 7 or later** for all new web development.
- Fetching a web page involves sending an **HTTP request** and receiving an **HTTP response**.
- ASP.NET Core allows **dynamic response generation** for incoming requests.
- An ASP.NET Core application contains a **web server** that serves as the entry point for requests.
