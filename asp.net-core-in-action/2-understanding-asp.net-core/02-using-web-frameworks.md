# Using a Web Framework

## Static vs dynamic web applications
- It is possible to build a **static web application** without a framework.
- Static applications have limited capabilities.
- As soon as you want **security** or **dynamism**, difficulties arise.
- Original simplicity fades when trying to build complex applications.

## What web frameworks provide
- Just as desktop or mobile frameworks help build applications, ASP.NET Core makes web development faster, easier, and more secure.
- ASP.NET Core contains libraries for common tasks such as:
  - **Creating dynamically changing web pages**
  - **User authentication** (letting users log in)
  - **OAuth integration** (Facebook, Google login support)
  - **Common application structure** for maintainable code
  - **Configuration file reading**
  - **Serving image files**
  - **Request logging** for web apps

## Dynamic web pages
- **Dynamic web pages** are the key to modern web applications.
- Display different data based on the **current logged-in user**.
- Display **user-submitted content** or **personalized data**.
- Without a dynamic framework, login functionality and personalization are impossible.
- Examples: Amazon, eBay, Stack Overflow demonstrate the power of dynamic frameworks.
- Web frameworks for creating dynamic pages are almost as old as the web itself.
- Microsoft has created several frameworks over the years.

## Why create a new framework?
- Microsoft felt the need to create **ASP.NET Core**, a new framework for modern web development.
